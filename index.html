1. How does React's virtual DOM reconciliation work?
React creates a lightweight in-memory representation of the real DOM. When state changes, a new virtual DOM is created. React uses a diffing algorithm to compare it with the previous snapshot, identifying the exact changes (reconciliation) and updates only those elements in the real DOM to optimize performance.
2. What is the difference between JWT and Session authentication?
JWT is stateless; the user info is encoded in a token stored on the client side, signed by the server. Sessions are stateful; user session state is stored on the server (in memory/DB), and a session ID is stored in a cookie. JWT is excellent for microservices and scalable stateless APIs, while sessions offer easier token revocation.
3. How do you design middleware to secure Express.js routes?
You create a custom middleware function that intercepts incoming requests, extracts the JWT from the Authorization header (e.g., 'Bearer <token>'), verifies it using the server secret via jwt.verify(), and attaches the decoded payload to the request object before calling next() to grant access to the route.
